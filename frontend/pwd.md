Project 2: React App
Individual project.
Deadline: 1st of April
Project Overview

You must build a React-based web application that allows users to manage a collection of items.
Users must be able to:
Add new items (doesn’t have to persist on refresh)
View items in a list
Edit existing items
Delete items

The type of data is flexible (e.g. books, recipes, movies, songs, travel destinations), but the application must demonstrate core React concepts and good application structure.

The goal is to create a dynamic, interactive, and user-friendly interface using React.
Requirements
Item Management
Users must be able to add a new item
Users must be able to view all items in a list or grid
Users must be able to edit an existing item
Users must be able to delete an item
Changes must be reflected immediately in the UI (no page reloads)

User Interface
The application must have a clear and intuitive layout
UI elements (buttons, inputs, lists) must be easy to understand and use
Feedback should be provided for user actions (e.g. item added, item deleted)

React Usage
The application must be built using React
JSX must be used to render UI elements
Components must be used to structure the application
Use either an external API or your own that you built in the previous course.
Assessment Criteria – For PASS
To achieve a Pass, the application must meet all of the following:

Component-Based Architecture
The application must be split into multiple React components
Example: App, ItemList, ItemCard, ItemForm
Each component should have a clear responsibility
Components should be reusable where appropriate

State and Interactivity
React state must be used to manage application data
State updates must trigger UI updates
Props must be used to pass data between components

Dynamic Behavior
The application must respond to user input dynamically
No hard-coded item lists in JSX
Data must be stored in state (not static HTML)

Code Quality
Code must be readable and consistently formatted
Components and variables should have meaningful names
No unused or dead code
Assessment Criteria – For PASS WITH DISTINCTION (PWD)
To achieve a Pass with Distinction, the application must demonstrate clear evidence of scalability, structure, and thoughtful design.

Scalable Application Structure
Components are well-organized (e.g. folders for components)
Logic is separated from presentation where appropriate
The structure makes it easy to add new features in the future

Effective State Management
State is lifted only when necessary
Components manage only the state they need
Updates are predictable and easy to follow
No unnecessary re-renders or duplicated state

Thoughtful Data Handling
Forms include basic validation (e.g. required fields)
Item data is handled consistently across the app
Edge cases are considered (empty list, cancelling edits, etc.)

User Experience and Polish
Smooth, responsive interactions
Clear visual hierarchy and spacing
UI feels intentional and user-focused
Errors or invalid input are handled gracefully

Maintainability
Code is easy to understand for another developer
Components are small and focused
Logic is reusable and not duplicated
Submission Expectations
A working React application.
Application must run without errors.
Clear demonstration of React fundamentals.
Source code submitted via GitHub repository.
Live URL.
Brief presentation max 5 min.
